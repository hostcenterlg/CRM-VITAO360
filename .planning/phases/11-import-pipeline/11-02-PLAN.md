**scripts/motor/classify.py:**

**Funcao 1: classificar_registros(dfs: dict[str, pd.DataFrame]) -> dict[str, pd.DataFrame]**

Aplica classificacao 3-tier a cada registro de cada DataFrame:

Regras de classificacao por aba:
- **CARTEIRA**: REAL (dados vem de INDEX/MATCH das DRAFTs e OPERACIONAL -- sao dados recalculados de fontes primarias SAP/Mercos)
- **OPERACIONAL**: REAL (fonte SAP direta)
- **DRAFT 1**: REAL (staging Mercos -- fonte primaria)
- **DRAFT 2**: Classificar por origem:
  - Se coluna de origem indica "CONTROLE_FUNIL" e o registro esta na lista dos 558 -> ALUCINACAO
  - Se coluna de origem indica Mercos/SAP/Deskrio -> REAL
  - Demais -> SINTETICO (com flag)
- **DRAFT 3**: REAL (staging SAP -- fonte primaria)
- **SINALEIRO**: SINTETICO (calculado a partir de dados REAL)
- **MOTOR DE REGRAS**: REAL (definicao de regras, nao dado)
- **REGRAS**: REAL (definicao de regras)
- **Abas consultor** (LARISSA/MANU/JULIO/DAIANE): Classificar por conteudo:
  - Registros com data e CNPJ valido -> REAL
  - Registros dos 558 ALUCINACAO (match por CNPJ + data se disponivel) -> ALUCINACAO
  - Demais -> SINTETICO

Para identificar os 558 registros ALUCINACAO:
- Ler aba DRAFT 2 (que contem o LOG operacional)
- Procurar registros que vieram do CONTROLE_FUNIL e foram previamente catalogados como fabricados
- Criterios de deteccao: registros do CONTROLE_FUNIL que tem padroes de dados fabricados pelo ChatGPT (datas muito regulares, valores redondos demais, CNPJs que nao existem no SAP/Mercos)
- ALTERNATIVA PRAGMATICA: Se nao conseguir identificar precisamente os 558 por criterio automatico, marcar TODOS os registros do CONTROLE_FUNIL como SUSPEITO e logar para revisao. O importante eh NAO integrar dados nao validados.

Adicionar coluna 'classificacao_3tier' a cada DataFrame com valor "REAL", "SINTETICO" ou "ALUCINACAO".
