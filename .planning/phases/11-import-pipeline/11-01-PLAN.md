Criar o pipeline de importacao principal:

**scripts/motor/import_pipeline.py:**

Funcao principal: importar_planilha(caminho=None) -> dict[str, pd.DataFrame]

**Estrategia de leitura:**
- Usar openpyxl com data_only=True para ler valores calculados (nao formulas)
- NAO usar pandas read_excel diretamente (nao suporta data_only)
- Fluxo: openpyxl load_workbook -> iterar rows -> construir DataFrame

**Implementacao detalhada:**

1. Abrir workbook:
   ```python
   wb = openpyxl.load_workbook(caminho or CAMINHO_PLANILHA, data_only=True, read_only=True)
   ```
   - read_only=True para performance (6.2 MB, 210K formulas)
   - data_only=True para obter valores calculados

2. Para cada aba em ABAS_RELEVANTES + ABAS_CONSULTOR:
   - Usar safe_read_sheet(wb, nome_aba)
   - Se None -> logar warning e pular
   - Extrair headers da primeira row com dados (row 1 para maioria, CUIDADO com abas que tem headers em rows diferentes)
   - Converter para DataFrame com pd.DataFrame(data, columns=headers)

3. Tratamento especial por aba:
   - **CARTEIRA** (1593 rows x 144 cols): Header na row 3 (rows 1-2 sao super-grupo headers). Ler a partir da row 4 como dados. Colunas A-EN conforme radiografia_carteira.json. CNPJ esta na coluna B.
   - **OPERACIONAL** (663 rows x 23 cols): Header na row 1 (padrao). CNPJ na coluna W (formatado).
   - **DRAFT 1** (566 rows x 60 cols): Header na row 3 (rows 1-2 sao agrupamento). CNPJ na coluna BH.
   - **DRAFT 2** (1001 rows x 40 cols): Header na row 1. Sem CNPJ direto (usa Nome Fantasia).
   - **DRAFT 3** (1542 rows x 18 cols): Header na row 1. CNPJ na coluna R.
   - **SINALEIRO** (665 rows x 26 cols): Header na row 1.
   - **MOTOR DE REGRAS** (98 rows x 12 cols): Header na row 1. Tabela de 92 combinacoes.
   - **REGRAS** (496 rows x 13 cols): Header na row 1. 25 modulos + tbl_MotorV2.
   - **Abas consultor** (LARISSA/MANU/JULIO/DAIANE): Header na row 1. ~13159 rows x 40 cols de LOG.

4. Normalizar CNPJs em TODAS as abas que tem coluna CNPJ:
   - Aplicar normalizar_cnpj() na coluna de CNPJ de cada DataFrame
   - Adicionar coluna 'cnpj_normalizado' como chave padronizada
   - Logar quantos CNPJs eram None/invalidos antes vs depois

5. Normalizar vendedores na CARTEIRA:
   - Coluna K = CONSULTOR -> aplicar normalizar_vendedor()
   - Adicionar coluna 'consultor_normalizado'
   - Logar distribuicao: quantos por vendedor canonico

6. Retornar dict com chaves = nomes_logicos (mesmas chaves de ABAS_RELEVANTES + consultor names), valores = DataFrames

7. Imprimir resumo ao final:
   ```
   === IMPORT PIPELINE COMPLETO ===
   Abas lidas: 19/19
   CARTEIRA: 1590 rows (excl. headers)
   OPERACIONAL: 660 rows
   ...
   CNPJs normalizados: X total, Y unicos, Z nulos
   Vendedores: MANU=N, LARISSA=N, DAIANE=N, JULIO=N, LEGADO=N, DESCONHECIDO=N
   ```

**CUIDADO com data_only=True:**
- Celulas que nunca foram calculadas (arquivo nunca aberto no Excel) retornam None
- Se a planilha FINAL foi salva pelo Excel (como eh o caso), os valores calculados DEVEM estar presentes
- Se uma coluna inteira retorna None, logar ALERTA e sugerir abrir no Excel, salvar, e reimportar

**Performance:**
- read_only=True eh essencial -- sem isso, openpyxl tenta carregar tudo em memoria
- Para a CARTEIRA (180K formulas), esperar ~30-60 segundos de leitura
- Logar tempo de leitura por aba
