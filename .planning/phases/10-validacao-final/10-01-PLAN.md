  <action>
Create `scripts/phase10_validacao_final/01_comprehensive_audit.py` that performs a full audit of the V13 workbook.

**Python path:** `/c/Users/User/.pyenv/pyenv-win/pyenv-win/versions/3.12.10/python.exe`
**V13 path:** `data/output/crm/CRM_VITAO360_V13_PROJECAO.xlsx`

**Structure:**
1. **Accent-stripping helper** (unicodedata normalize) for PROJECAO sheet name with cedilla
2. **Two-pass loading:**
   - Pass 1: `data_only=False` for formula text analysis (VAL-01)
   - Pass 2: `data_only=True, read_only=True` for cached value analysis (VAL-03, VAL-04)
3. **Print progress** per tab (file is 5.1 MB, takes 2-5 min to load)

**VAL-01: Formula Error Scan (all 13 tabs)**
- Iterate ALL cells in ALL 13 tabs (use `data_only=False` workbook)
- Check formula text (strings starting with `=`) for error patterns: `#REF!`, `#DIV/0!`, `#VALUE!`, `#NAME?`, `#N/A`, `#NULL!`
- Also check for dangerous patterns: `_xlfn.LET` (unsupported in older Excel), unbounded column refs like `$A:$A` without row bounds
- For each error found: record tab name, cell coordinate, error type, formula excerpt (first 100 chars)
- Count total formulas per tab and compare against expected (PROJECAO ~19,224; CARTEIRA ~134,092; etc.)
- Verdict: PASS if 0 error patterns found, FAIL otherwise

**VAL-02: Faturamento Structural Integrity**
- Cannot validate computed totals (openpyxl can't recalculate) -- validate STRUCTURE instead
- Check DRAFT 1 monthly columns (U through AF) have numeric data for 554 clients
- Check CARTEIRA FATURAMENTO block (cols BZ-JC, rows 4-557) contains formulas referencing DRAFT 1
- Sample 10 FATURAMENTO formulas: confirm they reference `'DRAFT 1'!` with correct column letters
- Check REALIZADO formula in PROJECAO col Z: should be `=SUM(AA:AL)` pattern (bounded or unbounded)
- Verdict: PASS_WITH_NOTES (structure correct, PAINEL R$ 2,156,179 mismatch is pre-existing Phase 2 FAIL_WITH_NOTES -- document reference)

**VAL-03: Two-Base Architecture (LOG)**
- Use `data_only=True` workbook (cached values)
- Iterate LOG tab starting row 3 (row 2 is headers)
- For each record: check ALL numeric cells in columns 5+ (index 5 onward) for non-zero values
- A violation = any non-zero float/int in monetary-relevant columns
- Count total records and violations
- Verdict: PASS if 0 violations, FAIL otherwise

**VAL-04: CNPJ Validation**
- Extract CNPJs from: DRAFT 1 col B (start row 4), CARTEIRA col B (start row 4), LOG col D (start row 3)
- Format check: strip non-digits, must have exactly 14 digits, not all zeros
- Duplicate check: within DRAFT 1, within CARTEIRA (LOG can have duplicates -- one per interaction)
- Cross-reference: DRAFT 1 set should match CARTEIRA set exactly (554 clients each)
- Verdict: PASS if 0 format errors AND 0 duplicates in DRAFT 1/CARTEIRA

**VAL-05: Tab Inventory (13 tabs -- updated from ROADMAP's 14)**
- List all sheet names from workbook
- Compare against expected 13: PROJECAO (with cedilla), LOG, DASH, REDES_FRANQUIAS_v2, COMITE, REGRAS, DRAFT 1, DRAFT 2, CARTEIRA, AGENDA LARISSA, AGENDA DAIANE, AGENDA MANU, AGENDA JULIO
- Use accent-stripping for PROJECAO comparison
- For each tab: report row count, column count, formula count
- Verdict: PASS if all 13 present (note: ROADMAP says 14, actual V13 has 13 -- document discrepancy as requirement update)

**Cross-Tab Reference Validation (bonus check):**
- For all formula cells, extract sheet references (regex for `'sheetname'!` pattern)
- Verify each referenced sheet exists in the workbook
- Report any orphaned references
