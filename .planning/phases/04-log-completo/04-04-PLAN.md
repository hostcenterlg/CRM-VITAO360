**PASSO 3 — Validacao das 15 regras obrigatorias:**

Executar TODAS as validacoes do RESEARCH.md "Validation Rules Summary":

1. **Two-Base Architecture:** Verificar que NENHUM registro tem campos com valores monetarios (R$, valor > 0 em campos nao-data). Assertar PASS/FAIL.
2. **CNPJ Format:** Todos CNPJs exatamente 14 digitos, zero-padded, sem pontuacao.
3. **No Weekends:** Todos DATA.weekday() < 5. Contar violacoes.
4. **Max 40/day/consultant:** GROUP BY (CONSULTOR, DATA), verificar COUNT <= 40. Listar violacoes.
5. **VENDA needs ORCAMENTO:** Para cada RESULTADO='VENDA / PEDIDO', verificar que mesmo CNPJ tem RESULTADO='ORCAMENTO' com DATA entre D-1 e D-3 business days. Contar falhas.
6. **PROSPECT needs CADASTRO:** Para VENDA de PROSPECT, verificar CADASTRO antes. Contar falhas.
7. **No duplicate dedup key:** Confirmar zero duplicatas apos dedup.
8. **Julio Gadret normalized:** Somente 1 grafia de JULIO GADRET.
9. **Helder timeline:** Zero registros Helder apos 2025-08-31.
10. **Julio timeline:** Zero registros Julio antes 2025-09-01.
11. **Monthly distribution:** Comparar com targets (JAN:156..DEZ:796), calcular desvio %.
12. **Note variety:** Contar frequencia de cada nota, listar as que excedem 3x.
13. **MOTIVO only when needed:** MOTIVO so preenchido quando RESULTADO e nao-venda.
14. **LIGACAO ATENDIDA logic:** Se LIGACAO=NAO entao LIG.ATENDIDA=N/A.
15. **Total count >= 11,758:** Assertar contagem final.

Para cada validacao: PASS/FAIL + contagem de violacoes + detalhes das primeiras 5 violacoes.
