**1. Funcao `generate_journey_for_sale(cnpj, sale_date, situacao, consultor, rede, uf, nome)`:**

Para cada venda SAP, verificar se ja tem registros correspondentes no CONTROLE_FUNIL (por CNPJ + mes). Se SIM: verificar se falta ORCAMENTO/CADASTRO/MKT obrigatorios e gerar somente os faltantes. Se NAO: gerar jornada completa.

Jornadas conforme GENOMA COMERCIAL:
- **Jornada A (PROSPECT):** 6-10 contatos. Sequencia: EM ATENDIMENTO iniciais -> FOLLOW UPs -> ORCAMENTO (D-1 a D-3) -> CADASTRO (antes do ORCAMENTO) -> VENDA (D0) -> MKT (D+2/3) -> SUPORTE passivo (30%) -> CS (D+7/10)
- **Jornada B (ATIVO):** 3-5 contatos. Sequencia: [EM ATENDIMENTO] -> ORCAMENTO (D-1 a D-3) -> VENDA (D0) -> MKT (D+2/3) -> CS
- **Jornada C (INAT.REC/EM RISCO):** 5-8 contatos. Sequencia: EM ATEND -> [NAO RESP] -> [FOLLOW UP] -> ORCAMENTO -> VENDA -> MKT
- **Jornada D (INAT.ANT):** 5-8 contatos. Sequencia similar a C com mais follow-ups
- NAO ATENDE/NAO RESPONDE intercalados aleatoriamente (10-15% das jornadas)

**Variacoes humanizadas (OBRIGATORIO):**
- Quantidade de contatos: normal distribution (mean=tipo.media, std=2, min=3, max=15)
- Offsets entre contatos: +-1 dia do padrao
- 10% das jornadas tem "surpresas": follow-up extra, nao-atende no meio
- 5-10% dos campos opcionais (MOTIVO, MERCOS ATUALIZADO) ficam vazios
- Channels via generate_channels() do _helpers.py

**Regras CRITICAS:**
- Calendario: Somente seg-sex via dia_util() e subtract_business_days()
- Capacidade: Max 40/dia/consultor via capacity tracker
- Timeline: Respeitar CONSULTANT_WINDOWS
- TODO VENDA DEVE ter ORCAMENTO antes (D-1 a D-3 mesmo CNPJ)
- TODO PROSPECT com VENDA DEVE ter CADASTRO antes
