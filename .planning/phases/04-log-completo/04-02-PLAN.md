**Passo 2 — Construir ETL baseado na estrutura real encontrada:**

O script `02_process_deskrio.py` deve:

1. **Ler fonte principal** (provavel 07_TICKETS_DESKRIO.xlsx ou CONTATOS DESKRIO.xlsx):
   - Mapear colunas dinamicamente por nome de header
   - Extrair: data do ticket/conversa, CNPJ do cliente (se disponivel), nome do cliente, assunto/resumo, status, consultor/atendente

2. **Se CNPJ nao disponivel diretamente:**
   - Tentar match por NOME FANTASIA contra roster de clientes existente (carregar `data/output/phase03/population_report.json` ou `data/output/phase02/sap_mercos_merged.json` para obter lista de CNPJs conhecidos)
   - Se match encontrado: usar CNPJ do match
   - Se nao: marcar como 'CNPJ_PENDENTE' e excluir do JSON final (com contador)

3. **Agregar ao nivel de CONVERSA (nao mensagem):**
   - Se dados estao ao nivel de mensagem: agrupar por (ticket_id ou conversa_id), pegar a primeira data como DATA, o ultimo status como RESULTADO
   - Se dados ja estao ao nivel de ticket: usar diretamente (1 ticket = 1 registro LOG)
   - Alvo: ~5,425 conversas, NAO 77,805 mensagens

4. **Mapear para LOG 20 colunas:**
   - DATA: data da conversa/ticket (normalizar para YYYY-MM-DD)
   - CONSULTOR: atendente Deskrio -> mapear para nomes padrao via normalize_consultor()
   - RESULTADO: mapear status do ticket para os 12 resultados padrao:
     * Resolvido/Fechado → SUPORTE
     * Em andamento → EM ATENDIMENTO
     * Aguardando → FOLLOW UP 7
     * Novo → EM ATENDIMENTO
     * Cancelado → PERDA / FECHOU LOJA
   - TIPO ACAO: Deskrio = RECEPTIVO (cliente inicia o chat)
   - TIPO DO CONTATO: "CONTATOS PASSIVO / SUPORTE" (maioria sao suporte via chat)
   - NOTA DO DIA: Resumo do ticket/conversa (assunto + primeiras linhas da mensagem)
   - WHATSAPP: NAO (Deskrio e chat proprio, nao WhatsApp)
   - LIGACAO: NAO
   - LIGACAO ATENDIDA: N/A
   - Campos derivados: via motor_de_regras()

5. **Validacoes:**
   - Descartar registros sem DATA valida
   - Descartar registros em sabado/domingo
   - Zero valores financeiros
   - CNPJs normalizados para 14 digitos

6. **Salvar JSON:** `data/output/phase04/deskrio_normalized.json`:
   ```json
   {
     "metadata": {"source": "DESKRIO", "total": N, "cnpj_matched": N, "cnpj_pendente": N, "weekends_skipped": N, "date_processed": "2026-02-17"},
     "records": [...]
   }
   ```

7. **Print estatisticas:** total processados, com CNPJ, sem CNPJ, por mes, por consultor.
