Criar `scripts/phase06_ecommerce/01_extract_ecommerce.py` que:

1. **Descobre todos os arquivos e-commerce** em 2 localizacoes:
   - `data/sources/mercos/ecommerce/` (16 .xlsx + 2 .xls)
   - `data/sources/mercos/fev2026/acesso ao ecomerce fevereiro 2026.xlsx`
   Ignorar arquivos temporarios (~$*).

2. **Le cada .xlsx com header detection dinamica:**
   - `find_header_row(ws)`: procura "Razao Social" ou "Raz" na col 1 para encontrar o header (tipicamente row 6, nao row 1).
   - `get_emission_date(ws)`: extrai data de emissao do relatorio (row 3 tipicamente: "Emitido em DD/MM/YYYY").
   - `detect_format(ws, header_row)`: detecta formato pela presenca de "E-mail" na coluna 3:
     * OLD_9COL: Razao Social, Nome Fantasia, [7 data cols]
     * NEW_11COL: Razao Social, Nome Fantasia, E-mail, Telefone, [7 data cols] (offset +2)
   - Para cada data row apos header: extrair razao_social, nome_fantasia, acessos, itens_adicionados, orcamentos_finalizados, atividades, valor_carrinho, valor_orc_nao_finalizados, valor_pedidos_b2b.
   - Extrair CNPJ do prefixo do nome em formato novo: regex `^(\d[\d.]+)\s` no inicio da Razao Social.
   - Usar `safe_int()` e `safe_float()` para converter valores (tratar None, '', strings).

3. **Para os 2 arquivos .xls:** Tentar import xlrd. Se disponivel, ler com xlrd (mesmo header detection). Se nao disponivel, logar warning e skip (nao bloquear o ETL). Instalar xlrd via pip antes de tentar.

4. **Dedup em 2 niveis:**
   a. **Dedup de arquivos identicos**: Comparar primeiros 5 registros (razao_social uppercase[:30] + acessos) entre todos os arquivos. Arquivos com mesma signature -> manter o de mais rows, descartar restantes. Documentar cada decisao.
   b. **Dedup de meses**: Quando 2 arquivos cobrem o mesmo mes (mesma emission_date ou conteudo sobreposto), preferir o arquivo com mais rows. Documentar.
   - Regras especificas da pesquisa:
     * Trio Abril/Maio/junho (com espaco): identicos -> manter 1 so, marcar mes como "DESCONHECIDO" se nao bater com adjacentes
     * "Acesso Dezembro " e "Acessos Dezembro " (24 rows cada): identicos -> descartar ambos em favor de "Dezembro 2025" (108 rows)
     * "janeiro 2026" (66 rows) vs "rELATORIO JANEIRO 2026" (134 rows): preferir rELATORIO (mais rows, emitido mais tarde). Opcionalmente, verificar se sao sets disjuntos para potencial merge.
     * "b2b fevereiro 2026" (50 rows) vs "fevereiro 2026" do fev2026/ (98 rows): preferir fev2026/ (mais dados)

5. **Mapear mes real** usando emission_date (nao nome do arquivo!):
   - "Emitido em 15/12/2025" com 44 rows = provavelmente Marco (posicao no calendario)
   - Usar heuristica: se emission_date anterior ao suposto mes, considerar o nome do arquivo como pista secundaria.
   - Armazenar `assigned_month` no formato "YYYY-MM" para cada arquivo.
