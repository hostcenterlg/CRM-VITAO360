Criar `scripts/phase06_ecommerce/02_match_populate.py` que:

1. **Carregar ecommerce_raw.json** (output do plan 06-01). Extrair monthly_data.

2. **Construir lookup bidirecional nome->CNPJ** usando 3 fontes:
   a. **Mercos Carteira** (`data/sources/mercos/08_CARTEIRA_MERCOS.xlsx`, sheet "Carteira Clientes Mercos"):
      - Col A = Razao Social, Col B = Nome Fantasia, Col C = CNPJ
      - Criar lookup: uppercase(razao_social) -> cnpj E uppercase(nome_fantasia) -> cnpj
      - ~497 entries
   b. **DRAFT 1** (`data/sources/crm-versoes/v11-v12/DL_DRAFT1_FEV2026.xlsx`):
      - Col 3 (C) = CNPJ, Col 4 (D) = Razao Social (ou Nome Fantasia -- verificar headers)
      - Rows 4 ate max_row com dados
      - ~554 entries
   c. **sap_mercos_merged.json** (`data/output/phase02/sap_mercos_merged.json`):
      - Chaves sao CNPJs (537 unicos)
      - Sem nomes direto, mas usar para validar CNPJs encontrados

   Reutilizar padrao de `build_mercos_lookup()` de `scripts/phase02_faturamento/04_fuzzy_match_sem_cnpj.py`.

3. **Matching em 4 niveis** (para cada registro de e-commerce):
   a. **CNPJ do prefixo**: Se `cnpj_from_name` preenchido e >= 11 digitos, usar direto (normalize_cnpj + zfill(14))
   b. **Exact match**: uppercase(razao_social) no lookup
   c. **Exact match por Nome Fantasia**: uppercase(nome_fantasia) no lookup
   d. **Partial match**: um nome contido no outro (min 5 chars, preferir match mais longo)
   e. **Fuzzy match (opcional)**: Se rapidfuzz disponivel, threshold 85. Se nao disponivel, skip.

   Para cada match, registrar: cnpj, match_level (cnpj_prefix/exact/partial/fuzzy), match_detail.
   Para unmatched, registrar razao_social + nome_fantasia + reason.

4. **Agregar por CNPJ** apos matching:
   Para cada CNPJ com dados de e-commerce, calcular:
   - `acessa_ecommerce`: true se qualquer mes teve acessos > 0
   - `data_ult_acesso`: max(assigned_month) entre meses com acessos > 0 (formato "YYYY-MM-DD", usar ultimo dia do mes)
   - `qtd_acessos_ultimo_mes`: acessos do mes mais recente com dados
   - `pedido_via_ecommerce`: true se qualquer mes teve valor_pedidos_b2b > 0
   - `valor_pedidos_b2b_total`: sum de valor_pedidos_b2b across all months
   - `catalogo_visualizado`: true se qualquer mes teve atividades > 0
   - `meses_com_acesso`: lista de meses (YYYY-MM) em que o cliente acessou
   - `total_atividades`: sum de atividades across all months
   - `total_itens_carrinho`: sum de itens_adicionados
   - `total_orcamentos`: sum de orcamentos_finalizados
   - `valor_carrinho_total`: sum de valor_carrinho
   - `monthly_data`: dict month -> {acessos, itens, orcamentos, atividades, valor_carrinho, valor_orc_nf, valor_b2b}

   **Para % PEDIDOS E-COMMERCE:** Carregar sap_mercos_merged.json e calcular:
   - fat_total = sum dos 12 meses de vendas do CNPJ
   - Se fat_total > 0: pct_ecommerce = valor_pedidos_b2b_total / fat_total
   - Se fat_total == 0: pct_ecommerce = null (nao calcular)
